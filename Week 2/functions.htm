Functions

In JavaScript, groups of statements that you can execute as a single unit are called functions. Functions are useful because they make it possible to treat a related group of JavaScript statements as a single unit. Functions, like all JavaScript code, must be contained within a <script> element.

Defining Functions

Before you can use a function in a JavaScript program, you must first create, or declare, it. The lines that make up a function are called the function definition. The syntax for defining a function is:

function functionName(arguments) {
      statements;
}

Arguments or parameters are placed within the parentheses that follow a function name. An argument is a variable that is used within a function. Placing an argument name within the parentheses of a function definition is the equivalent of declaring a new variable. Functions can contain none or multiple arguments separated by commas.

Following the parentheses that contain the function arguments is a set of curly braces that contain the function statements. Function statements are the statements that do the actual work of the function and must be contained within the function braces.

Example: Assume you want to create a function that takes the total sales figure and determines the grand total which includes the sales tax. The following statements create such a function.

 

<script type="text/javascript">
        function printName(total){
                 var salesTax =.06;
                 var grandTotal= salesTax*total;
                 document.write("The total is: " + grandTotal);
           }
</script>

 

Type the above statements inside an HTML document. What is the result?  If you answered nothing, you are correct. Next you'll learn how to actually invoke the function.

 

 

Calling Functions

A function definition does not execute automatically. Creating a function definition only names the function, specifies its parameters, and organizes the statements it will execute. To execute a function, you must invoke, or call, it from elsewhere in your program. The code that calls a function is referred to as a function call and consists of the function name followed by parentheses that contain any variables or values to be assigned to the function parameters. The variables or values that you place in the parentheses of the function call statement are called arguments or actual parameters. Sending arguments to the parameters of a called function is called passing arguments.

Example: Create a function that writes message Hello Students to the web page.

....
<script type="text/javascript">
      function practice() {
         document.write("Hello Students");
      }
</script>
</head>
<body>
 <script type="text/javascript">
            practice();
</script>

 

In the above example, the function contains no arguments so the parentheses after the function name is empty.

Note: Always put your functions within the document head, and place calls to a function within the body section. As you recall, the document head is always rendered before the document body. If your program does attempt to call a function before it has been created, you will receive an error.

Now, lets assume we want to create a function that takes the user's first name as an argument, and displays a welcome message using the visitor's name.

 

............
<head>
 <script type="text/javascript">
           function welcome(visitorName) {
               document.write("Hello " + visitorName);
            }   
</script>
</head>
<body>
<script type="text/javascript">
            var firstName= prompt("Enter your first name:");
            welcome(firstName);
</script>
</body>

Result

 

Note that in the above example, the function is accepting one argument named "visitorName" which is the firstName retrieved through the prompt box. Also note that even thought the variable names used in this example are not the same (firtName and visitorName,) they represent the same value which is the first name the visitor types in the prompt box.

Returning a value

Functions can calculate a value and send it to the calling statement. To do so, you need to assign the calling statement to a variable.   In order to send (return) a value to the calling statement, you need to use the return statement followed by the name of the variable you want to return. A function does not necessarily have to return a value.

Example: Assume you want to create a function that requires two numbers. The function should calculate the average of the two numbers and return the result to the calling statement.

 

.................
<head>
 <script type="text/javascript">
            function average(num1, num2) {
                var result=(num1 + num2)/2;
                return result;
            }
</script>
</head>
<body>
<script type="text/javascript">
            var first=5;
            var second=20;
            var summary=average(first, second);
            document.write(summary);
</script>
</body>

 

Result

Note that the average function is returning the result to the calling statement, and the calling statement is assigning the result to a variable named "summary".

Variable Scope

 A variable's scope can either be local or global. A global variable is declared outside a function and can be used and referenced throughout the program. The following shows an example of a global variable that is used within the function:

 

 <script type="text/javascript">
            var num1=5;
</script>

 

The var keyword is optional when you declare a global variable.


A local variable is declared inside a function and can be used and referenced only within that function.  If you try to reference a local variable outside that function, you will receive an error message. The following shows the improper use of a local variable:

 

 .................
<head>
 <script type="text/javascript">
            function example() {
               var num1=10;
            }
</script>
</head>
<body>
<script type="text/javascript">
            example();            
            document.write(num1);
</script>

 Copy and paste the code in your text editor and note that the code will result in an error message.

If you declare a variable within a function's scope and don't include the var keyword, then that variable becomes a global variable.
