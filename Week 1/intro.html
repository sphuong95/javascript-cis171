Introduction to JavaScript

The original Internet was planned and developed in the early 1960s. At first it was accessible only by the government and academic institutions. By the end of the 1980s, access was expanded to business users. The development of the hypertext link in the early 1990s was crucial to the development of the portion of the Internet known as the World Wide Web. Documents on the Web, or Web pages, are accessed by using a Web browser such as Microsoft Internet Explorer. In the mid 1990s, the World Wide Web Consortium, or W3C, was established to develop standards for Web programming. Web pages are written in hypertext markup language, or HTML. This language was originally designed to specify the logical structure of a document without information about presentation. However, it evolved into a combination of both. HTML documents can be created in a text editor such as Notepad, or in a program such as Microsoft FrontPage. The advantage of using a program designed to create HTML documents is that you do not need to know all of the tags that make up the document.

Initially, documents that were displayed on the Web were written in HTML. This language was first developed in the early 1990s. HTML documents mix logical content with presentation information. For this reason, HTML is not appropriate for use with a new generation of user agents, such as PDAs and cell phones. It is being replaced with XHTML, a language that is similar but has stricter rules. The W3C has introduced three document type definitions, or DTDs, that support the XHTML standard. A DTD is a specification of the tags and structure that are allowed in a document. A document may be validated against a DTD. This means that the document format is checked against the rules defined in the DTD to determine whether it complies with those rules. The rules enforced by XHTML are fairly simple, for example, all empty elements must be closed and all attribute values must be enclosed in double-quotes. The W3C provides a free XHTML validation service.

JavaScript is a client-side scripting language that allows Web page authors to develop interactive Web pages and sites. Originally designed for use in Navigator Web browsers, JavaScript is now also used in most Web browsers including Internet Explorer. Client-side scripting refers to a scripting language that runs on a local browser (the client) instead of on a Web server, which is a special type of computer used for hosting Web sites. The part of a browser that executes scripting language code is called the browser's scripting engine. A JavaScript program is written within an HTML or XHTML document by using the <script> element. This element tells the Web browser that the scripting engine should interpret the commands it contains. The individual lines of code, or statements, that make up a JavaScript program in a document are contained within the beginning and ending <script> elements. In addition to being an interpreted scripting language, JavaScript is considered an object-based programming language.

JavaScript elements can be placed in different locations in a Web page. They can be placed inside any element or in the head or body of the document. It is a good idea to enclose the contents of a script in comment tags to prevent browsers that do not execute scripts from displaying the script text in the document. JavaScript can also be placed in a separate script file. This is a good option for longer scripts. It also enables script code to be shared among multiple Web pages. The src attribute of the <script> element is used to reference the script file.
