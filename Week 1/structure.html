Introduction

The <script> tag is used to create JavaScript tags inside a web page. The type attribute which specifies the type of information included in the script should equal to text/javascript. You include the following code in a document to tell Web browser the statements that follow must be interpreted by JavaScript scripting engine:

<script type="text/javascript">
    Statements                
</script>                      

JavaScript is a case sensitive language. Using the wrong case for an object or method will generate an error message. 


Comments in JavaScript

Comments are programming remarks that a programmers inserts into a program's source. The comments that you add to your JavaScript statements have no effect on your codes, and the JavaScript interpreter ignores them completely. JavaScript supports two types of comments:  

  •   // for a single line comment  
     

  • /*    
    . . . .                                                                            
    */    for multi-line comments (block comments) 

     

Example:

<script type="text/javascript">    
     // Pam Barvarz
</script>

<script>
          /* Pam  Barvarz
             OCC
             Fall 2011  */    </script>

Writing Valid JS Code

If you were to validate an XHTML document containing a script section, the document would not validate successfully. The reason is that in an XHTML document, the statements in a <script> element are treated as parsed character data (PCDATA,) which identifies a section of a document that is interpreted as markup. As a result, if you attempt to validate an XHTML document that contains a script section, it might fail the validation.

To solve this issue, you have two options:

  1. Moving the JavaScript code into a source file.

  2. Enclosing the code within a <script> element within a CDATA section, which marks sections of a document as character data, or CDATA. Any sections marked as CDATA will not be interpreted as markup. To mark your codes as CDATA, you need to use the following syntax:

    <![CDATA[
          JavaScript statements
    ]]>

When the browser detects the script tags, it executes the statements one line after the other assuming all the statements are JavaScript statements. When the JavaScript interpreter gets to the <! CDATA[    and    ]]> codes it will detect those as errors. To fix this issue you should mark the <! CDATA[  and  ]]> portions of the CDATA section as JavaScript comments as shown below:

<script type="text/javascript">
     /*  <![CDATA[  */
         JavaScript Statemets
     /* ]]> */
</script>                                       

JavaScript is considered an object-based programming language.  An object is  a programming code and data that can be treated as individual unit or component.  An example of an object is the document object.  A method is an action that can be performed with or on an object.  An example of a method is the write() method.  You can use the write() method to write contents to the screen. Method ids are listed after the object ids, and every method id is followed by a pair of parentheses. Inside the parentheses, you need to insert arguments. Arguments are the information that the method needs. For example, the write() method requires a text string as its argument. 

The only reason to use the write() method is to add new text to a Web page while it is being rendered.  Remember that the write()  method require a text string as an argument.

Example:             

 

<script type="text/javascript">
     document.write("Welcome to OCC!");
</script>  

As shown in the example, a JavaScript statement is terminated with a semicolon.

Structure of JavaScript

Scripts can be added to a Web page in one of three places:

  1. External file (source file). A JavaScript source file is usually designated by the file extension .js and contains only JavaScript statements; it does not contain a <script> element. JavaScript source files cannot include XHTML or HTML elements.

  2. Head section

  3. Body section

Each has advantages and disadvantages. Scripts placed in an external file can be easily reused on multiple pages. If those pages are all part of the same site, download time is decreased because the scripts only download when the first page requests them. After that, they're stored in the browser's cache. The disadvantage of external scripts is that any modifications must be tested on all the Web pages that use them.

Scripts placed in the head section are loaded before the content in the <body> section, meaning they are guaranteed to be present, when called from inside the body. When scripts are placed in the head section, they are easier to find when navigating code because they're all in one location.

The body section is the least preferred location for scripts. The only advantage to body scripts is that their code is usually near the location where they are called, making code tracing a little easier. One disadvantage of body scripts is that a script may not finish loading before the viewer triggers some event that calls the scripts. If that happens, the page usually crashes. Another disadvantage is that body scripts can be hard to locate within large page files. Both body and head scripts download with their pages, so any code that's repeated across multiple pages will be downloaded every time it shows up instead of being cached. This means that download size and time for the pages will be larger than if the scripts were placed in external files.

Remember that when including multiple script sections in a document, you must include a <script> element for each section.

LOGIC AND DEBUGGING

Every language (including JavaScript) has its own rules, called syntax. Logic refers to the order in which various parts of a program run, or execute. You need to use the correct syntax and follow the correct logic in order for your programs to work correctly. If you have an incorrect syntax or you did not follow the logic correctly, you end up with errors, also called bugs.

Important Notes about JavaScript

Because JavaScript is an interpreted language, it is more forgiving than compiled languages such as C++ or Java. Even though it is flexible, JavaScript has a set of syntax rules that must be obeyed. The most basic rules are:

  1. JavaScript is case-sensitive and most of its reserved words are in lowercase.

  2. All strings must be enclosed in quotations.

  3. JavaScript commands end with a semicolon, just like C++ and Java.

Unfortunately, you cannot always rely on the line number.  Sometimes the errors come from the previous lines of your codes, but checking the line number is the start in finding your errors.
