Inline JavaScript

Inline JavaScript refers to JavaScript statements that are used within an HTML tag such as an anchor or image tags. Inline JavaScript allows you to integrate a JavaScript command into a single HTML tag.  When you use inline JavaScript, you should not use the <script> tag.

Events

An event is a specific circumstance (such as an action performed by a user, or an action performed by the browser) that is monitored by JavaScript and that your script can respond to in some way. You can use JavaScript events to allow users to interact with your Web pages. The most common events are actions that users perform.  For example, when a user clicks a form button, a click event is generated. You will learn about different types of JavaScript events in the future weeks.

When an event occurs, your script executes the code that responds to that particular event. Code that executes in response to a specific event is called an event handler. You include event handler code as an attribute of the element that initiates the event. For example, you can add to a <button> element a click attribute that is assigned some sort of JavaScript code, such as code that changes the color of some portion of a Web page.

Event handler ids are the same as the id of the event itself, plus a prefix of "on". For example, the event handler for the click event is onclick. JavaScript event handlers can appear as HTML tag attributes and don't require the <script> tags.

Example:  Create buttons that change the background color of a web page using the onclick event handler. To change the background color of a web page, you need to use the bgColor attribute of the document object.

<input type="button" value="black" onclick="document.bgColor='black'; "  />

<input type="button" value="red" onclick="document.bgColor='red'; "  />

<input type="button" value="yellow" onclick="document.bgColor='yellow'; " />

<input type="button" value="pink" onclick="document.bgColor='pink'; "  />

<input type="button" value="white" onclick="document.bgColor='white'; "  />

Note: Pay particular attention to the syntax of an inline script.  The entire script is enclosed in double quotes.  If a string is going to be displayed as part of the JavaScript code; for example, the color id, that string is enclosed in single quotes.  The string must be closed BEFORE the quotation mark that marks the end of the event handler.  Getting the punctuation out of order is a common problem among even experienced programmers.

alert Dialogue Boxes

At times it might be necessary to notify the user of possible issues or problems with their input. The alert() method allows you to show messages to the users in a dialogue box. The alert method is an integral part of letting the user know of errors found in their input.

The syntax of the alert method is as follows:

alert("Message");

 

Example: Create an alert box that displays the message "Welcome to My Web Page".

<script type="text/javascript">
      alert("Welcome to My Web Page");
</script>

Type the above statement in a Web page. What do you see? Discuss your observations in Unit 1 forum of the discussion board.

You can also write the alert() method as an inline JavaScript. For example, you can create an anchor tag that displays a Welcome message when the use clicks on the hypertext Click Here for a Welcome Message.

<a href="#"
onclick="alert('Welcome to My Page');">
   Click Here for a Welcome Message.
</a>

 

Result:

                                        Click Here for a Welcome Message.

Functions

In JavaScript, groups of statements that you can execute as a single unit are called functions. Functions are useful because they make it possible to treat a related group of JavaScript statements as a single unit.  Functions, like all JavaScript code, must be contained within a <script> element.

Before you can use a function in a JavaScript program, you must first create, or declare, it. The lines that make up a function are called the function definition. The syntax for defining a function is:

function functionid() {
    statements;
}

Following the parentheses  is a set of curly braces that contain the function statements. Function statements are the statements that do the actual work of the function and must be contained within the function braces.

A function definition does not execute automatically. Creating a function definition only ids the function and organizes the statements it will execute. To execute a function, you must invoke, or call, it from elsewhere in your program. The code that calls a function is referred to as a function call and consists of the function id followed by parentheses.
